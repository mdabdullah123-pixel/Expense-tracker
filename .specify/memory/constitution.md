# Project Constitution

Purpose
-------
This document defines the governance, goals, and rules for Spec-Driven Development in this repository.

Principles
----------
- Open collaboration: all contributors follow the spec-first workflow.
- Clarity: specifications are written in plain language and map to tests/implementation.
- Traceability: every feature spec links to a plan and tasks.

Governance
----------
- Owners: repository maintainers review and approve specs.
- Acceptance: a feature is accepted when its spec, plan, and tasks are merged and tests pass.

Revision
--------
This constitution may be amended by maintainers; changes should be recorded in this file.
